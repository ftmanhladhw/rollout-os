# Rollout OS Manifesto

See conversation version. This document defines the guiding principles:

- Rollouts are different from BAU
- Context before tasks
- Visibility should be automatic
- One operational workspace
- Leadership needs understanding
- Reference documents, don't duplicate
- Opinionated defaults
- Simplicity
- AI augments judgement
- Knowledge survives people
- Temporary by design
- Every feature must improve rollout success
